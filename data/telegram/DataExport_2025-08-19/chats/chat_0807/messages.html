Hello! üëã Great to connect.

In essence, our product helps two parties to enter into special multisigs (DLCs) on Bitcoin. However, some crucial details of this contract has to be stored 'off-chain' (from the POV of bitcoin). As long as either party has this info, they can finalize their contract. However, if both parties lose these details, the DLC remains closed forever.

Currently, we are storing these details for both parties in a regular database on AWS. The data itself is not "sensitive", although the more privacy we can provide, the better (but exposing the info would not allow an attacker to do anything malicious). Nevertheless, we don't really like how centralized this solution is. So we are exploring decentralized storage solutions, which is how we found WeaveVM.

So, in short: users would need to write/read information from a storage, multiple times/day (depending on usage), preferably with minimal fee overhead. The users would not have to know about this solution in the background.

Let us know if this is something that could work, and if you have any specific questions! Thanks!

cc. @j_eisenberg
