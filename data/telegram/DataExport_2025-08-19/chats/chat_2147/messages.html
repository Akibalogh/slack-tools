Joe Hoffend WalletConnect Reown
To start it will just be for connecting bitcoin wallets and allowing for signing (see more below). AppKit can detect injected wallets when the user doesn't want to use a mobile wallet. Eventually we can add other features we currently have for other chains such as OnRamp, Swaps, etc but for Bitcoin based on demand.

There is an updated canonical RPC spec we worked on with several Bitcoin wallets to account for Taproot and signing needs. We support it as well. Can take a look here: https://docs.reown.com/advanced/multichain/rpc-reference/bitcoin-rpc

It supports taproot and importantly, "signMessage" which is a huge need for the upcoming "Travel Rule". We are doing DevRel with many other Bitcoin wallets to get them supporting this RPC spec. You can learn about why it's so important here, which is a doc I wrote thay we've been sharing with Wallets to educate them on travel rule: https://www.notion.so/walletconnect/Reown-and-the-Travel-Rule-an-opportunity-for-Wallets-1213a661771e807d89dcf3183a53d0b1

Lastly and important, we are building out deeper user insights tools on the Cloud backend we have. This will allow you for example to see the user's btc address and then we will layer in onchain insights such as "Average spending power of your users" or anything else you may want to see (what competitors your users be trending to use more too). So kind of a blend of CRM + Analytics and user insights
