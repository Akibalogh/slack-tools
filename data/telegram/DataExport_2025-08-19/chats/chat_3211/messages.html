For now, what I can share is that it's absolutely our goal to decentralize the network. For example, our customers should not fear that we will collude with a hacker, or that we would run off with the attestor keys.

However, it's still unclear whether a token is the right solution for this. Alternative solutions exist, such as running attestors on Chainlink's oracle network via Chainlink Functions. Or, attestors could be run within HSMs in partnership with BitGo or Coinbase.

The main difference is, unlike oracle networks where node runners can come and go, the attestors must stay live to be able to close the DLC. If a number of third-party attestors unexpectedly leave, the DLC might 1. not be able to close, or 2. may timeout and close in an inconvenient way, leading to poor UX. So it's unclear what the token model is - would we want to completely slash a node runner for leaving?
