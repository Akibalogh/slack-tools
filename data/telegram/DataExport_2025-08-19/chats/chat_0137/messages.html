Hi Alex - happy to clarify.

Yes, a DLC is a 2-of-2 multisig UTXO where the user is pre-signing 2 transactions

DLCs run on mainnet. They use Schnorr and PTLCs, and they are already supported on Bitcoin.

But DLC logic is extremely difficult to implement. The DLC signing behavior has to be implemented into wallets, apps and into the attestors.

We provide the infrastructure to run whitelabel DLCs for any app. You can see the components here:
https://www.dlc.link/blog/components-of-dlc-link

This makes it easy for any protocol to implement DLCs. So we plug into Eth apps, then will move to Solana, Polkadot, BSC, etc. These protocols gain the ability to take Bitcoin deposits and move native Bitcoin with their smart contract logic
