Hey I just thought of something. Sorry if you guys have already considered it but why not use a blockchain (or a custom one) to manage the consensus and then use on-chain multi-sig instead of threshold algorithm to coordinate attestations?

XRP ledger supports escrow which you can execute using a set of multisig keys. It might be heavyweight for you to run on your own so you could either modify it to be lightweight, purpose built for your use-case or build something from scratch using cosmos SDK or Avalanche.

The multi-sig is really nice because you can quite easily add/remove keys. Rotations etc is all managed on-chain and you can probably, if you build your own blockchain write governance transactions for such operations.
