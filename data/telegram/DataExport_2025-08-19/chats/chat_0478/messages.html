Martin Krung - Curve DAO
just to recap our call:

1. lowest bar would be to make crvUSD available on canton, to be able to tx in privacy, we can look into that, but as crvUSD is very closely tied to our DeFi protocol, we need to think if there is any demand and how much integration work it will be.

2. stableswap, our core tech, would need a full rewrite, a fork. I'm not sure if an AMM would work, as the quote is generated by the on-chain code itself, and this needs some transparency. If at some point we think this can be done, we could license the tech to a 3 party, then these third party would build it with our help.
