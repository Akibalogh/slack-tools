Starting with information on how to set up a node for yourselves:

Start by making the request at this link: https://sync.global/validator-request/

There are not currently unlimited nodes (parties can multi-tennat with operators if they don't feel passionately about having a separately allocated node), but new ones are being added on a regular basis. If you'd like to run nodes on behalf of others, you would be the one working to potentially offer multi-tenant access.

Once you've filled out the form at the link above, you'll begin the process outlined at a high level below, and covered in greater detail in "Onboarding to the Canton Network" here: https://docs.dev.sync.global/validator_operator/validator_onboarding.html

1. Prepare a container environment (either in a Virtual Machine or a Kubernetes cluster) in any cloud or your own data center
2. Set up a fixed egress IP address for the cluster that will run the node, and ask your sponsor to submit it for whitelisting (pending application approval)
3. Prepare the OIDC authentication setup (follow documentation to find the best route based on your enterprise‚Äôs authentication system, e.g.: Auth0)
4. Deploy your validator node on DevNet. Afterwards, in collaboration with your sponsor super validator, deploy on TestNet and MainNet
