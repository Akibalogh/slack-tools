I see what you mean. Yes that is the Attestors' job.

When the user locks BTC, they sign a Bitcoin TX with the Attestor Network. These same Attestors are responsible for signaling on the EVM side any updates. The user can verify the validity at any step of the process.

On the Bitcoin side, Attestors use FROST signatures to create a threshold signature, on the EVM side, it's a regular ECDSA multisig.

To avoid race conditions, Attestors are 'stateless' as much as possible - they merely react to user requests, and do the necessary verification on the respective chains, with the necessary security delays built in.

- user creates vault
- submits BTC tx to Attestors
- Attestors verify, wait, signal to EVM, mint the token
- user burns on EVM, submits withdraw tx
- Attestors verify, wait, etc....

Process and history is fully verifiable on both chains

I think the advantage of the system is it's simplicity!
